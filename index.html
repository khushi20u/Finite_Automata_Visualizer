📖 What is a Finite Automaton?
🧠

The Core Idea

A Finite Automaton (FA) is a mathematical model of computation — a simple machine that reads an input string one character at a time and decides whether to accept or reject it.

Think of it as a flowchart with a fixed set of "rooms" (states). You start in one room and follow arrows (transitions) based on what you read. At the end, if you're in a special "accepting room," the string is accepted.

🏗️

5 Formal Components

Every finite automaton is defined by a 5-tuple (Q, Σ, δ, q₀, F):

  • Q — Finite set of states
  • Σ — Alphabet (input symbols)
  • δ — Transition function
  • q₀ — Start state (q₀ ∈ Q)
  • F — Set of accept states (F ⊆ Q)
⚖️ DFA vs NFA — What's the Difference?
Property DFA — Deterministic NFA — Nondeterministic
Transitions Exactly one per (state, symbol) pair Zero, one, or many per (state, symbol)
Epsilon (ε) Not allowed Allowed — "free" transitions with no input consumed
Multiple paths No — single path through states Yes — explores all paths simultaneously (conceptually)
Acceptance Accept if final state is an accept state Accept if ANY possible path ends in an accept state
Computational power Exactly the same — every NFA can be converted to an equivalent DFA
Implementation Simpler, direct More expressive to design, subset construction for DFA conversion
Dead states Can reach a dead (trap) state → immediate reject Missing transition = that branch dies; other branches continue
🔵 DFA — Deterministic Finite Automaton
DFA

How DFA Works

In a DFA, every state must have exactly one transition for every symbol in the alphabet. There is no ambiguity — at any point in time, the machine is in exactly one state.

δ: Q × Σ → Q
(state, symbol) → next state

The transition function δ is total — defined for every possible combination. If you don't define a transition, it implicitly goes to a "dead" reject state.

DFA Example

Even Number of 0s

This DFA accepts all binary strings containing an even count of the digit 0:

  • q0 = start & accept (even 0s seen so far)
  • q1 = odd number of 0s seen
  • On 0: toggle between q0 and q1
  • On 1: stay in current state

Try it: 00110 → Accept · 010 → Reject

🟣 NFA — Nondeterministic Finite Automaton
NFA

How NFA Works

In an NFA, a state can have zero, one, or multiple transitions on the same symbol. Additionally, ε (epsilon) transitions allow jumping to states without consuming any input.

δ: Q × (Σ ∪ {ε}) → 2Q
(state, symbol) → set of states

The NFA "guesses" the right path. We say it accepts a string if at least one possible computation path leads to an accept state.

NFA Example

Strings Ending in "01"

This NFA accepts binary strings that end with "01":

  • q0 = start (loop on 0 and 1)
  • On 0 from q0 → also go to q1 (guess this is the last 0)
  • On 1 from q1 → go to q2 (accept)
  • q2 = accept state

The "split" on 0 is nondeterminism — both paths explored at once!

ε Epsilon (ε) Transitions Explained
⚡

What is an Epsilon Transition?

An epsilon (ε) transition — also called a "free move" or "lambda transition" — allows the NFA to move from one state to another without reading any input symbol. The input tape does not advance.

ε-closure(S)

The epsilon-closure of a set of states S is all states reachable from S by following zero or more epsilon transitions. When simulating an NFA, after every move (including at the start), we immediately compute the ε-closure of the current set of active states.

Why Are ε-transitions Useful?

  • They make NFA design much easier — think of them as "free shortcuts" in your machine
  • Simplify expressing alternation: "match pattern A or pattern B" becomes easy with ε-transitions to both sub-machines
  • They're the foundation of how regular expressions are compiled to automata (Thompson's construction)
  • Any NFA with ε-transitions can be converted to one without them (ε-removal algorithm)
Example: q0 --ε→ q1 means: whenever the machine is in q0,
it can simultaneously also be in q1 at no cost.
🎯 How to Build Your Own FA — Step by Step
1

Define the Language

Write out in plain English what strings your FA should accept. Example: "All binary strings with an odd number of 1s." Test your understanding by generating a few accept and reject examples by hand.

2

Choose DFA or NFA

Use a DFA when the pattern can be tracked with a clear finite count of "status" conditions. Use an NFA when the pattern is easier to express nondeterministically (e.g., "ending with XY" — NFA is more natural; you guess when you see the beginning of the suffix).

3

Identify Distinct "Memory" Conditions

Each state represents what the machine "remembers" about what it has seen so far. For "even number of 0s," you only need to remember whether the count is even or odd — so 2 states. Name and create those states.

4

Mark Start and Accept States

Exactly one state is the start state (the machine begins here). One or more states are accept states (being here when input ends means "accept"). In the Builder, click a state in the list to toggle accept, or right-click on the canvas for the context menu.

5

Add Transitions

For each state and each input symbol, decide where the machine should go. For DFA, every state needs a transition for every symbol in the alphabet. For NFA, transitions are optional — missing means "this path dies." Add ε-transitions to allow free moves between states.

6

Test with the Simulator

Enter strings you expect to be accepted and strings you expect to be rejected. Use the Step button to watch how the machine processes each character. Check the transition table to make sure every case is covered.

📐 Key Theorems to Know
Equivalence

DFA = NFA in Power

Despite their apparent differences, DFAs and NFAs recognize exactly the same class of languages: the Regular Languages.

The proof is constructive — using the Subset Construction (powerset construction) algorithm, any NFA with n states can be converted to a DFA with at most 2ⁿ states.

Closure Properties

Regular Languages are Closed Under

  • Union (L₁ ∪ L₂)
  • Concatenation (L₁ · L₂)
  • Kleene star (L*)
  • Complement (L̄)
  • Intersection (L₁ ∩ L₂)
  • Reversal (Lᴿ)
DFA Minimization

Minimizing a DFA

For every regular language there exists a unique minimal DFA (fewest states). The Myhill-Nerode theorem tells us the number of states needed equals the number of distinguishable equivalence classes of strings.

Use the table-filling algorithm to merge indistinguishable states.

Pumping Lemma

Proving Non-Regularity

Not all languages are regular! The Pumping Lemma gives a necessary condition for regular languages — if a language fails it, no FA can recognize it.

Example: {aⁿbⁿ | n ≥ 0} (equal a's and b's) is NOT regular — it requires infinite memory.

💡 Common Patterns & Tips

DFA Design Patterns

  • Counting mod n → needs n states arranged in a cycle
  • Ends with pattern P of length k → needs 2ᵏ states in worst case (DFA) or just k+1 states (NFA)
  • Contains substring → use KMP-like structure; needs |P|+1 states
  • Starts with prefix → linear chain of states for the prefix
  • Both A and B → product construction (|A| × |B| states)

NFA Design Tips

  • Use ε-transitions to compose machines (A then B: connect A's accept to B's start with ε)
  • For union of L₁ and L₂: new start state → ε to both machines' starts
  • For Kleene star L*: add ε from accept states back to start
  • For "suffix" patterns, NFA is simpler than DFA — guess when the suffix begins
  • Every regex has a direct NFA via Thompson's construction
